# 语音转发 + 审批分级 + 结果播报 — 产品设计方案

> 版本：v1.1 ｜ 日期：2026-06-23
> 更新：补充多会话多 Agent 设计、切换快捷键可配置

---

## 一、产品定位

DesktopAssistant 是 **Agent 传令官**，不是 Agent 本身。

核心价值：帮用户**操作**已有的编码 Agent（Claude Code、Codex、opencode 等），而不是替代它们。

```
用户 ←→ 桌面伴侣（传令官）←→ Claude Code / Codex / 其他 CLI Agent
         │
         ├─ 语音输入 → 转发指令给 Agent
         ├─ Agent 审批 → 按风险分级处理
         ├─ Agent 完成 → 结果播报给用户
         └─ 空闲时 → 陪聊 / 提醒 / 陪伴
```

## 二、核心功能

### F1: 语音转发（指令模式）

**当前问题**：语音输入只能跟角色聊天，无法把指令转发给正在运行的 Claude Code。

#### 两种模式

| 模式 | 触发 | 语音去向 | 气泡提示 |
|------|------|---------|---------|
| 💬 聊天模式（默认） | 启动时默认 | 角色本地 LLM 对话 | 💬 聊天模式 |
| 🔧 指令模式 | 按快捷键切换 | 转发给**当前选中的** Agent 会话 | 🔧 Claude Code (proj-a) |

#### 切换快捷键

- 默认 `Cmd+Shift+A`（macOS）/ `Ctrl+Shift+A`（Windows/Linux）
- **可在设置中自定义**，与语音输入快捷键（Alt+Space）使用相同的录制机制
- 按一次切换到指令模式，再按一次切回聊天模式
- 切换时角色播报当前模式和目标 Agent

#### 指令模式行为

1. 用户按 Alt+Space 说话 → ASR 识别文本
2. 查找当前选中的 Agent 会话
3. **有会话**：文本 + 回车 注入到 Agent 的 PTY stdin
4. **无会话**：角色提示"没有在运行的 Agent"，自动切回聊天模式

### F2: 按工具分级审批

**当前问题**：审批只有两档——黑名单（强制键盘）和其余全部需要语音确认。Read 文件这种无风险操作也要确认，打断开发节奏。

#### 四级审批策略

| 级别 | 行为 | 角色反应 | 适用场景 |
|------|------|---------|---------|
| 🟢 **auto** | 静默自动批准 | 无反应（不打扰） | 只读操作 |
| 🔵 **notify** | 自动批准 + 播报 | 气泡"已批准 Write" | 低风险写入 |
| 🟡 **confirm** | 语音提醒 + 等待确认 | "Claude 要执行 Bash，同意吗？" | 执行类操作 |
| 🔴 **block** | 语音警告 + 必须键盘确认 | "⚠️ 危险操作，请在终端确认！" | 破坏性操作 |

#### 默认规则

```
🟢 auto:    Read, Grep, Glob, LS, ListDir, SearchDir
🔵 notify:  Write, Edit, NotebookEdit, CreateFile
🟡 confirm: Bash, WebFetch, WebSearch, BrowserAction
🔴 block:   rm -rf, sudo, chmod 777, git push --force, DROP TABLE, ...
```

#### 审批流程图

```
Agent 请求审批 (tool = "Bash", prompt_text = "npm install express")
  │
  ▼
匹配 block_patterns？─── 是 ───→ 🔴 block: 语音警告 + 必须键盘
  │ 否
  ▼
tool 在 auto 列表？─── 是 ───→ 🟢 自动注入 y\r，静默
  │ 否
  ▼
tool 在 notify 列表？── 是 ───→ 🔵 自动注入 y\r，气泡+TTS "已批准 Edit"
  │ 否
  ▼
tool 在 confirm 列表？─ 是 ───→ 🟡 TTS "要执行 Bash，同意吗？" + 等语音
  │ 否
  ▼
未知工具 ──→ 🟡 默认走 confirm（安全优先）
```

#### 用户自定义

设置面板中提供规则编辑器：

```
┌─ 审批规则 ─────────────────────────────────┐
│                                              │
│  🟢 自动批准（静默）                          │
│  ┌────────────────────────────────────────┐  │
│  │ Read  Grep  Glob  LS  [+ 添加]        │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  🔵 自动批准（播报）                          │
│  ┌────────────────────────────────────────┐  │
│  │ Write  Edit  NotebookEdit  [+ 添加]    │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  🟡 语音确认                                  │
│  ┌────────────────────────────────────────┐  │
│  │ Bash  WebFetch  [+ 添加]               │  │
│  └────────────────────────────────────────┘  │
│                                              │
│  🔴 强制键盘（不可语音批准）                   │
│  ┌────────────────────────────────────────┐  │
│  │ rm -rf  sudo  chmod 777  [+ 添加]      │  │
│  └────────────────────────────────────────┘  │
│                                              │
```

### F3: 结果播报（三级可选）

**当前问题**：Claude Code 完成任务后，角色只说"任务完成啦"，不知道完成了什么。

#### 三个播报级别（用户在设置中选择）

| 级别 | 行为 | 角色说什么 | 延迟 |
|------|------|-----------|------|
| 🔇 **silent** | 只显示气泡，不语音 | 气泡：✅ 完成 | 0 |
| 🔔 **notify** | 语音通知完成 | "Claude 完成了任务" | ~1s |
| 📋 **summary** | 提取输出 → LLM 总结 → 语音播报 | "Claude 修改了 3 个文件，新增了登录验证功能" | ~3-5s |

#### summary 级别的工作原理

```
Agent 运行中
  ↓ PTY 输出持续写入 2000 字符缓冲区
  ↓
Agent 任务完成（Stop 事件）
  ↓ assist 二进制发送 TaskCompleted 事件 + output_tail
  ↓
桌面伴侣收到事件
  ↓ 把 output_tail 发给本地 LLM
  ↓ system: "用一句话总结这段 Agent 输出的工作内容，不超过 30 字"
  ↓
角色 TTS 播报 + cheer 动作 + happy 表情
```

#### 播报级别与接入方式的关系

| | Claude Code Hook 模式 | PTY 包装模式 (`assist run`) |
|---|---|---|
| 🔇 silent | ✅ | ✅ |
| 🔔 notify | ✅ | ✅ |
| 📋 summary | ❌ Hook 不携带输出 | ✅ PTY 有输出缓冲 |

设置面板提示：「摘要模式需要通过 `assist run` 启动 Agent」

## 三、多会话多 Agent 设计

### 3.1 会话注册模型

每个 Agent 会话在连接时注册自身信息：

```
┌─ AgentSession ──────────────────────────────┐
│  session_id    "claude-abc123"               │
│  agent_type    "claude-code"                 │
│  agent_label   "Claude Code"                 │
│  cwd           "/Users/xx/proj-a"            │
│  inject_url    "http://127.0.0.1:52341/inject" │
│  source        "hook" | "pty"                │
│  connected_at  1719100000000                 │
│  last_event_at 1719100300000                 │
└──────────────────────────────────────────────┘
```

**注册来源**：
- **Hook 模式**（Claude Code）：首次收到 Notification 事件时自动注册，从事件中提取 session_id / agent / cwd
- **PTY 模式**（assist run）：PTY 包装器启动时 POST `/pty/register` 主动注册，携带完整信息
- 两种来源统一存入同一个 `SessionRegistry`

**会话退出**：
- Hook 模式：收到 Stop 事件 → 标记会话为 `ended`，保留 5 分钟后清除（Claude Code 可能重连）
- PTY 模式：子进程退出时发 `/pty/unregister` → 立即清除
- 超时清除：30 分钟无事件 → 自动标记为 `stale` 并清除

### 3.2 多会话场景

同时可能存在多个 Agent 会话：

```
┌─ SessionRegistry ────────────────────────────────────────────────┐
│                                                                   │
│  #1  claude-abc  Claude Code    /Users/xx/proj-a     🟢 active   │
│  #2  claude-def  Claude Code    /Users/xx/proj-b     🟢 active   │
│  #3  codex-001   Codex          /Users/xx/proj-a     🟡 idle     │
│  #4  assist-xyz  opencode       /Users/xx/proj-c     🟢 active   │
│                                                                   │
│  当前选中: #1 claude-abc (Claude Code @ proj-a)                   │
│                                                                   │
└──────────────────────────────────────────────────────────────────┘
```

### 3.3 会话选择机制

**指令模式切换时**：

```
按 Cmd+Shift+A
  │
  ├─ 当前是聊天模式？
  │    ├─ 只有 1 个活跃会话 → 直接切到该会话
  │    ├─ 有多个活跃会话 → 显示会话选择菜单
  │    └─ 没有活跃会话 → 提示"没有在运行的 Agent"
  │
  └─ 当前是指令模式？
       └─ 切回聊天模式
```

**会话选择菜单**（多会话时弹出）：

```
┌─ 选择 Agent 会话 ───────────────────────┐
│                                           │
│  🟢 Claude Code    proj-a   3分钟前活跃   │
│  🟢 Claude Code    proj-b   刚刚活跃      │
│  🟡 Codex          proj-a   10分钟前      │
│  🟢 opencode       proj-c   1分钟前       │
│                                           │
│  按数字键快速选择 / ESC 取消              │
│                                           │
└───────────────────────────────────────────┘
```

选择后：
- 设置当前目标会话（`selectedSessionId`）
- 角色播报："已切换到 Claude Code，项目 proj-b"
- 气泡显示：🔧 Claude Code (proj-b)

**快捷切换（不弹菜单）**：
- `Cmd+Shift+A` 切换模式（聊天↔指令）
- `Cmd+Shift+1/2/3/4` 直接跳到第 N 个会话（如果有多个）
- 这些快捷键同样可在设置中自定义

### 3.4 审批事件与会话关联

审批请求自动路由到对应会话，无需用户选择：

```
Claude Code (proj-a) 请求审批 Bash
  ↓ 事件携带 session_id = "claude-abc"
  ↓
approvalRouter 记录: pending = { sessionId: "claude-abc", ... }
  ↓
用户语音"同意"
  ↓
inject("claude-abc", "y\r") → 自动路由到正确的 PTY/Hook 会话
```

审批不依赖"当前选中会话"——哪个会话发的审批请求，就回到哪个会话。多个会话同时有审批时，按到达顺序处理（先进先出队列，替代当前的单 slot 设计）。

### 3.5 审批队列（替代单 slot）

当前设计只有一个 `pending` slot，多会话时不够用。改为队列：

```
审批队列 (FIFO, 最多 10 条, TTL 10 分钟)
  ┌──────────────────────────────────────────┐
  │ #1 claude-abc  Bash "npm install"   2s前  │ ← 下一个处理
  │ #2 claude-def  Write "src/app.ts"  5s前   │
  │ #3 codex-001   Bash "jest"         8s前   │
  └──────────────────────────────────────────┘
```

语音确认时处理队列头部；auto/notify 级别自动出队处理不阻塞后续。

### 3.6 会话状态指示器

桌面角色旁边显示一个小型会话状态指示（可折叠）：

```
角色右上角小图标：

  无会话时：     (不显示)
  1个会话时：    🟢           ← 绿色小圆点
  多个会话时：   🟢②          ← 绿色 + 数字
  有待审批时：   🟡②          ← 黄色闪烁 + 数字
```

点击指示器展开简要面板：

```
┌─ Agent 会话 ──────────────────────┐
│  🟢 Claude Code   proj-a   ← 选中 │
│  🟢 Claude Code   proj-b          │
│  🟡 Codex (等待审批) proj-a        │
│                                    │
│  模式: 🔧 指令 → Claude (proj-a)   │
└────────────────────────────────────┘
```

## 五、与现有功能的关系

| 现有功能 | 影响 | 说明 |
|---------|------|------|
| 语音聊天 | 不变 | 聊天模式下行为完全不变 |
| 审批监控 | 增强 | 从二元变四级 + 审批队列替代单 slot |
| 黑名单拦截 | 保留 | block 级别 = 现有黑名单超集 |
| TTS 播报 | 增强 | 新增 auto/notify 级别差异化播报 |
| 行为引擎 | 扩展 | 新增 auto_approved/notify_approved 事件模板 |
| 设置面板 | 扩展 | 新增审批规则 + 播报级别 + 快捷键 + 会话面板 |
| PTY Registry | 重构 | 从 `HashMap<String, String>` 升级为 `SessionRegistry` |

## 六、技术约束

1. **Claude Code Hook 限制**：Stop hook 不携带输出内容，summary 播报仅 PTY 模式可用
2. **PTY 文本注入**：支持任意文本，特殊字符（Ctrl-C 等）需转义
3. **Hook 模式会话发现**：Claude Code 不主动注册，靠首次事件被动发现。会话可能在 Stop 后重连（保留 5 分钟窗口）
4. **Codex 兼容**：无 Hook 系统，只能通过 PTY 包装
5. **opencode**：通过 PTY 通用包装支持
