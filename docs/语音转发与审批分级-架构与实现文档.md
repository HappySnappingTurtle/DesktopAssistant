## 一、现有架构分析

### 1.1 当前数据流

```
语音输入 (Alt+Space → whisper ASR)
  ↓ voice://transcript 事件
approvalRouter.onTranscript(text)
  ├─ 有 pending 审批 → parseIntent → gate (黑名单/模式) → pty_inject(y/n)
  └─ 无 pending 审批 → onChat(text) → 本地 LLM 对话
```

### 1.2 现有关键数据结构

**PTY Registry**（`agent/http_server.rs:15`）：
```rust
pub struct PtyRegistry(pub Mutex<HashMap<String, String>>);
// session_id → inject_url，无 agent 名称、无 cwd、无时间戳
```

**Approval Slot**（`voice/approvalRouter.ts:7-10`）：
```typescript
interface PendingApproval {
  sessionId: string;
  promptText: string;
  at: number;
}
// 只有 1 个 slot，多会话审批会覆盖
```

**Security Gate**（`voice/securityGate.ts`）：
```
二元判定：BLACKLIST 命中 → block | 否则 → 由 ApprovalMode 决定
ApprovalMode: "auto"(禁止语音) | "safe-list"(允许语音) | "parrot"(只播报)
```

### 1.3 需要改造的瓶颈

| 组件 | 现状 | 目标 |
|------|------|------|
| PtyRegistry | 只存 inject_url | 升级为 SessionRegistry，存完整会话信息 |
| PendingApproval | 单 slot | 升级为 FIFO 队列（max 10） |
| SecurityGate | 二元（黑名单/通过） | 四级（auto/notify/confirm/block） |
| ApprovalRouter | 无路由模式 | 增加 chat/agent 双模式 |
| AgentEvent::TaskCompleted | 只有 summary 文本 | 增加 output_tail 可选字段 |
| 快捷键 | 只有 PTT 一个 | 增加模式切换快捷键 |

## 二、架构改造

### 2.1 SessionRegistry（替代 PtyRegistry）

**新数据结构**（Rust 侧）：

```rust
// agent/session.rs (新文件)

#[derive(Clone, Debug, Serialize, Deserialize)]
pub struct AgentSession {
    pub session_id: String,
    pub agent_type: String,       // "claude-code", "codex", "opencode", ...
    pub agent_label: String,      // 显示名 "Claude Code"
    pub cwd: String,
    pub inject_url: Option<String>, // PTY 模式才有
    pub source: SessionSource,
    pub connected_at: u64,
    pub last_event_at: u64,
    pub status: SessionStatus,
}

#[derive(Clone, Debug, Serialize, Deserialize)]
pub enum SessionSource { Hook, Pty }

#[derive(Clone, Debug, PartialEq, Serialize, Deserialize)]
pub enum SessionStatus { Active, Idle, Ended }

pub struct SessionRegistry(pub Mutex<SessionRegistryInner>);

pub struct SessionRegistryInner {
    sessions: HashMap<String, AgentSession>,
}
```

**Registry 操作**：

```rust
impl SessionRegistryInner {
    /// PTY 模式主动注册（assist run 启动时调用）
    pub fn register_pty(&mut self, session_id, agent_type, cwd, inject_url) -> &AgentSession;
    
    /// Hook 模式被动注册（首次收到事件时自动调用）
    pub fn register_or_touch_hook(&mut self, session_id, agent, cwd) -> &AgentSession;
    
    /// 更新最后活跃时间
    pub fn touch(&mut self, session_id: &str);
    
    /// 标记会话结束（保留 5 分钟后清除）
    pub fn mark_ended(&mut self, session_id: &str);
    
    /// 清理过期会话（ended 超 5 分钟 或 idle 超 30 分钟）
    pub fn gc(&mut self, now_ms: u64);
    
    /// 获取所有活跃会话（按 last_event_at 降序）
    pub fn active_sessions(&self) -> Vec<&AgentSession>;
    
    /// 查找会话
    pub fn get(&self, session_id: &str) -> Option<&AgentSession>;
    
    /// 获取 inject_url（审批注入用）
    pub fn inject_url(&self, session_id: &str) -> Option<String>;
}
```

**注册时机**：

| 来源 | 触发点 | 调用 |
|------|--------|------|
| PTY | `POST /pty/register` | `register_pty(session_id, agent_type, cwd, inject_url)` |
| Hook | 首次收到任意 AgentEvent | `register_or_touch_hook(session_id, agent, cwd)` |
| 后续事件 | 每次收到 AgentEvent | `touch(session_id)` |
| PTY 退出 | `POST /pty/unregister`（新端点） | `mark_ended(session_id)` |
| Hook Stop | 收到 TaskCompleted | `mark_ended(session_id)` |
| 定时器 | 每 60 秒 | `gc(now_ms)` |

**Tauri 命令**：

```rust
#[tauri::command]
pub fn list_agent_sessions() -> Vec<AgentSession>;

#[tauri::command] 
pub fn get_active_session_count() -> usize;
```

**兼容性**：现有 `pty_inject` 命令改为从 SessionRegistry 中查找 inject_url（原来从 PtyRegistry），签名不变。

### 2.2 审批队列（替代单 Slot）

**新数据结构**（TypeScript 侧）：

```typescript
// voice/approvalQueue.ts (新文件)

interface QueuedApproval {
  sessionId: string;
  agent: string;
  tool: string;
  promptText: string;
  at: number;
  level: ApprovalLevel; // 由 securityGate 判定
}

type ApprovalLevel = "auto" | "notify" | "confirm" | "block";
```

**队列行为**：

```
事件到达 → 判定 level
  ├─ auto   → 立即注入 y\r，不入队，不播报
  ├─ notify → 立即注入 y\r，不入队，气泡+可选TTS "已批准 {tool}"
  ├─ confirm → 入队尾，TTS 播报等待语音确认
  └─ block  → 入队尾(标记 blocked)，TTS 警告"请键盘确认"

语音"同意" → 取队列头部
  ├─ confirm → inject y\r，出队
  └─ block  → 拒绝，提示"这个操作必须键盘确认"
  
语音"不行" → 取队列头部 → inject n\r，出队

队列满(10条) → 丢弃最旧的 confirm 级别条目
TTL 10 分钟 → 过期条目自动移除
```

### 2.3 四级安全门（替代二元黑名单）

**改造文件**：`voice/securityGate.ts`

**现有接口（保持不变）**：
```typescript
function gate(input: GateInput): GateResult;
```

**新增查表逻辑**：

```typescript
// 审批规则配置（从 config 读取）
interface ApprovalRules {
  auto: string[];       // 工具名列表
  notify: string[];
  confirm: string[];
  block_patterns: string[];  // 正则模式
}

// 判定优先级：block_patterns > 精确匹配 > 默认 confirm
function resolveLevel(tool: string, promptText: string, rules: ApprovalRules): ApprovalLevel {
  // 1. block_patterns 命中 prompt_text → block
  if (rules.block_patterns.some(p => new RegExp(p, 'i').test(normalizeForScan(promptText)))) {
    return "block";
  }
  // 2. 工具名精确匹配
  if (rules.auto.includes(tool)) return "auto";
  if (rules.notify.includes(tool)) return "notify";
  if (rules.confirm.includes(tool)) return "confirm";
  // 3. 未知工具 → confirm（安全优先）
  return "confirm";
}
```

**gate() 改造**：

```typescript
function gate(input: GateInput & { tool: string; rules: ApprovalRules }): GateResult & { level: ApprovalLevel } {
  const level = resolveLevel(input.tool, input.promptText, input.rules);
  
  if (level === "auto") {
    return { allow: true, keys: "y\r", level: "auto" };
  }
  if (level === "notify") {
    return { allow: true, keys: "y\r", level: "notify" };
  }
  if (level === "block") {
    return { allow: false, reason: "blacklisted", level: "block",
             speech: "这个操作有风险，请你亲自在终端确认" };
  }
  // confirm 级别 → 走现有逻辑（等语音）
  // ... 现有 intent 判定逻辑
}
```

### 2.4 路由模式（聊天/指令切换）

**状态管理**（`voice/approvalRouter.ts` 改造）：

```typescript
interface RouterState {
  routeMode: "chat" | "agent";
  selectedSessionId: string | null;
}
```

**RouterDeps 扩展**：

```typescript
interface RouterDeps {
  inject: (sessionId: string, keys: string) => Promise<void>;
  onChat: (text: string) => void;
  onFeedback: (speech: string, mood: string) => void;
  onForwardToAgent: (sessionId: string, text: string) => Promise<void>; // 新增
  mode: () => ApprovalMode;
  rules: () => ApprovalRules;            // 新增
  getActiveSessions: () => AgentSession[];  // 新增
}
```

**onTranscript 改造**：

```typescript
async onTranscript(text: string) {
  // 1. 审批队列有 confirm 级别的待处理项？
  const pendingConfirm = queue.peek(q => q.level === "confirm");
  if (pendingConfirm) {
    // 走审批流程（与现有逻辑类似）
    return handleApproval(pendingConfirm, text);
  }

  // 2. 根据路由模式分发
  if (state.routeMode === "agent" && state.selectedSessionId) {
    await deps.onForwardToAgent(state.selectedSessionId, text);
    deps.onFeedback(`已发送给 Agent`, "neutral");
  } else {
    deps.onChat(text);
  }
}
```

**关键设计**：审批优先于路由模式——即使在指令模式下，如果有 confirm 级别的审批等待，语音仍然先处理审批。这保证了安全审批不会被路由模式绕过。

    case "notify":

## 六、风险与约束

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| Hook 会话无 inject_url，指令模式无法转发 | Claude Code Hook 模式不支持语音转发 | UI 提示"此会话不支持语音转发，请通过 assist run 启动" |
| auto 级别误批高危操作 | 安全风险 | block_patterns 优先级最高，匹配 prompt_text 而非 tool 名 |
| 多会话审批并发 | 用户困惑 | 队列按 FIFO 处理，TTS 播报包含 agent 名和项目名 |
| LLM 摘要质量差或超时 | summary 播报不准确 | 30 字上限 + 5s 超时 fallback 到 notify 级别 |
| PTY 注入多行文本丢字符 | 指令不完整 | 注入前转义特殊字符，限制单次注入 500 字符 |
